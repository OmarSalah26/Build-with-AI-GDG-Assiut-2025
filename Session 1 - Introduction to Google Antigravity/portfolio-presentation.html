🔄 Workflow

Step-by-Step Build Process

  • 1

    Define Your Goal & Audience

    Who reads this? Recruiters, clients, or the academic community? Your answer shapes everything.

  • 2

    Gather Raw Content

    Collect your CV, project links, certificates, photos, and GitHub repos in one folder first.

  • 3

    Use Gemini to Polish Copy

    Paste raw notes → prompt Gemini to rewrite as concise, engaging portfolio content.

  • 4

    Choose an Antigravity Template

    Pick a template for your field. Set brand colors and fonts before adding any content.

  • 5

    Build Section by Section

    Hero → About → Skills → Projects → Experience → Contact. Finish one before the next.

  • 6

    Review, Publish & Share

    Preview mobile + desktop → fix spacing → Publish → share on @iabdelrhmaneyad & LinkedIn.

🎨 Layout

Page Layout Anatomy

HERO — Name · Role · CTA Button
ABOUT ME — 3–4 sentences max
SKILLS — Tags or progress bars
PROJECTS — Cards (3-column grid)
EXPERIENCE — Timeline layout
CONTACT — Form + Social Links
⬛

Visual Hierarchy Rule

Name biggest → Role medium → Body small. Max 3 font sizes per section.

↕️

White Space Is Content

Min 80 px between sections. Breathing room signals professionalism.

📌

Sticky Navigation

Fixed nav at top — any section reachable in one click.

🎨

2-Color Accent System

1 primary + 1 accent + neutral base. More than 3 colors = noise.

✍️ Content

How to Organise Your Content

Hero Section

  • Full name + role headline
  • One-line value proposition
  • CTA: "View Projects" button
  • Secondary CTA: Download CV
  • Social icons — @iabdelrhmaneyad

About Me Section

  • 3–4 sentence professional story
  • Current role: TA at AI-BUA
  • Research & teaching interests
  • One personal fact (memorable)
  • Professional headshot — right-aligned

Projects Section

  • Feature only 3–6 best projects
  • Card: thumbnail · title · 2-line desc
  • Tech stack badges (Python, TF…)
  • Links: GitHub repo + Live demo
  • Sort: most recent or most impressive

Skills & Contact

  • Group: AI/ML · Tools · Languages
  • Use tags/pills — avoid vague bars
  • Contact: email + Calendly link
  • Repeat social handles in footer
  • Add a brief availability note

✅ Best Practices

Do's & Don'ts

✅ Do

→ Keep to one scrollable page (max 4 sub-pages)
→ Use real project screenshots & live demos
→ Include measurable outcomes ("accuracy +18%")
→ Test on mobile before publishing
→ Update at least once per semester
→ Link your @iabdelrhmaneyad prominently

❌ Don't

→ Don't list every project — curate ruthlessly
→ Don't use clip-art or generic stock photos
→ Don't publish AI content without reviewing it
→ Don't use more than 2 font families
→ Don't leave placeholder "Lorem ipsum…" text
→ Don't forget alt-text on images (accessibility)
