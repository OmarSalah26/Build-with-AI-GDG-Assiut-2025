## User Scenarios & Testing *(mandatory)*

### User Story 1 - Evaluate Resume against Job Description (Priority: P1)

As a recruiter or hiring manager, I want to upload a candidate's resume (PDF) and a job description (TXT) so that I can see how well the candidate matches the job requirements based on an AI-generated score.

**Why this priority**: This is the core functionality and main value proposition of the tool.

**Independent Test**: Can be fully tested by providing a sample PDF and TXT file to the system and verifying that a numerical score is returned based on the AI evaluation.

**Acceptance Scenarios**:

1. **Given** a valid PDF resume and valid TXT job description, **When** the user triggers the scoring process, **Then** the system extracts text from the PDF, sends both texts for evaluation, and returns a matching score.
2. **Given** an invalid or corrupted PDF file, **When** the user triggers the scoring process, **Then** the system displays a clear error message indicating the file cannot be read.

---

### Edge Cases

- What happens when the uploaded PDF consists of scanned images without embedded text?
- How does the system handle cases where the AI service is unavailable or returns a timeout error?
- What happens if the combined text of the resume and job description exceeds the maximum context length allowed by the AI service?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST accept a PDF file upload for the candidate's resume.
- **FR-002**: System MUST accept a text file upload or text input for the job description.
- **FR-003**: System MUST extract text content from the uploaded PDF resume.
- **FR-004**: System MUST send the extracted resume text and job description to Google Gemini for evaluation.
- **FR-005**: System MUST display the resulting score and detailed reasoning and feedback alongside the numerical score.
- **FR-006**: System MUST handle errors during PDF parsing gracefully.
- **FR-007**: System MUST provide the user interface via a Web Application (UI).
- **FR-008**: System MUST process scanned PDFs by rejecting them with a clear error.

### Key Entities

- **Resume**: Represents the candidate's uploaded document and its extracted text content.
- **JobDescription**: Represents the requirements text against which the resume is evaluated.
- **EvaluationResult**: Represents the final score and (if applicable) the detailed feedback generated by the AI.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: System successfully extracts text from standard text-based PDFs with >95% accuracy.
- **SC-002**: System returns an evaluation score within 15 seconds of submission on average.
- **SC-003**: Users can complete the full process from file upload to receiving a score in less than 30 seconds.

## Assumptions

- Users will provide their own API key or the system is configured with a valid organizational key.
- Resumes are standard text-based PDFs.
- The total text size of the resume and job description is well within the context limits.
- The evaluation score will be a numerical value (e.g., 0-100 or 1-10).
